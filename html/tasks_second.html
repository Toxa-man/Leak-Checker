Лаб. работа №1

Разработать класс реализующий линейный список и протестировать его функционал

Требования к классу:
Реализовать методы:
List::List() //дефолтный конструктор
List::List(unsigned size) //создаёт список размером size и заполняет его дефолтными значениями, т.е. T()
List::List(usingned size, const T& val) //создаёт список размером size и заполняет его значением val
unsigned List::size() //возвращает размер списока
void List::resize(unsined size) // приравнивает размера списока к size; если новый размер больше старого - новые элементы заполнить нулями
bool List::insert(unsigned pos, const T& val) //вставляет val в позицию pos; возвращает true | false в зависимости от успеха операции
void List::push_back(const T& val) //добавляет val в конец списока
bool List::contains(const T& val) //возвращает true если val присуствует в списоке и false в противном случае
??? List::operator [](unsinged index) // обращение к элементу с индексом index. При этом должно быть возможно как читать значение, так и изменять его
int List::find(const T& val) //возвращает индекс первого вхождения val в список или -1 в случае его отсутствия
А так же все необходимые функции для предотвращения проблем с памятью: copy-constructor, assigment operator, destructor

Примечания:
List - название для примера, вы можете создать класс с другим именем
??? - вы должны сами придумать какой тип использовать
T - алиас для типа, который вы задаёте сами перед объявлением класса используя следующий синтаксис: using T = int; где вместо int можно использовать любой другой тип

Лаб. работа №2

Необходимо разработать программу, которая принимает на вход файл и определяет, является этот файл валидным PNG изображением или нет.
Критерии валидности PNG изображения необходимо найти самим в официальной спецификации (доступна в интернете), либо из иных источников.
Для проверки корректности работы вашей программы прикрепляется демонстрационая версия . Вместе с ней в папке img находятся тестовые файлы изображений.
Поскольку кейсов валидности файла существует очень много, вам разрешается обрабатывать только те, которые обрабаываются в демонстрационной программе.
Требования к выводу программы:
1) Наличие сообщений о невалидности файла вместе с причиной
2) Для каждого IDAT блока необходимо вывести его позицию в файле и значение CRC (см. вывод демонстрационной программы)
3) В случае если файл является валидным - сообщение об этом
